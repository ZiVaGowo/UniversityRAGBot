## 📖 О проекте 

**UniversityRAGBot** — это AI-ассистент, который помогает студентам и абитуриентам быстро находить информацию в официальных документах университета.

### 🎯 Цель проекта

Создать систему, которая:
1. **Автоматически** загружает и обрабатывает PDF-документы
2. **Понимает** смысл вопросов (а не просто ищет слова)
3. **Отвечает** только на основе официальных документов (без галлюцинаций!)
4. **Указывает** источники (страницу и название документа)

### 🧠 Логика работы
1. ***Пользователь задаёт вопрос***
2. ***Вопрос превращается в вектор (числа) с помощью модели эмбеддингов*** 
3. ***Ищем похожие по смыслу куски текста в PDF (FAISS)***
4. ***Найденные куски + вопрос отправляем в GROQ (LLM)***
5. ***GROQ генерирует ответ ТОЛЬКО на основе найденных документов*** 
6. ***Пользователь получает ответ с указанием источников***

### Возможности бота

| Возможность | Описание |
|-------------|----------|
| Загрузка PDF | Поддерживает любые PDF с текстовым слоем |
| Умный поиск | Ищет по смыслу, а не по ключевым словам |
| Генерация ответов | Отвечает на естественном языке |
| Цитирование | Указывает название документа и страницу |
| Быстрый запуск | Запускается в 2 команды |

# Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/ваш-username/UniversityRAGBot.git
cd UniversityRAGBot

```
### 2. Создать виртуальное окружение
```bash
# Windows
python -m venv .venv
.venv\Scripts\activate

# Linux/Mac
python3 -m venv .venv
source .venv/bin/activate
```
### 3. Установить зависимости
```bash
pip install -r requirements.txt
```
### 4. Настроить .env
```bash
cp .env.example .env
# Открыть .env и добавить GROQ_API_KEY
.env.example:

env
GROQ_API_KEY=ваш_ключ_сюда
GROQ_MODEL=llama-3.1-8b-instant
```

### 5. Получить GROQ API ключ
1. ***Зарегистрироваться на console.groq.com***
2. ***Перейти в "API Keys"***
3. ***Создать новый ключ***
4. ***Скопировать и вставить в .env***

### 6. Добавить PDF в data/
Положите ваши PDF-документы в папку data/

### 7. Запустить проект
```bash
python main.py
```
### Терминал 2 (Streamlit UI):
```bash
streamlit run streamlit_app.py
```
### 8. Открыть в браузере(на самом деле не обязательно )
API документация: http://localhost:8000/docs

Интерфейс: http://localhost:8501

# Как это работает

### Этап 1: Загрузка документов (Ingestion)
1. Скрипт сканирует папку data/ и находит все PDF

2. Для каждого PDF извлекается текст с сохранением номеров страниц

3. Текст разбивается на чанки (куски) по 500-800 символов

4. Для каждого чанка создаётся эмбеддинг (векторное представление)

5. Все эмбеддинги сохраняются в FAISS индекс для быстрого поиска

### Этап 2: Поиск (Retrieval)
1. Пользователь задаёт вопрос

2. Вопрос превращается в эмбеддинг

3. FAISS находит топ-5 самых похожих чанков

4. Найденные чанки сортируются по релевантности

### Этап 3: Генерация ответа (Generation)
1. Найденные чанки + вопрос отправляются в GROQ API

2. GROQ генерирует ответ ТОЛЬКО на основе предоставленного контекста

3. Ответ возвращается пользователю с указанием источников

# Как добавить свои документы

## Просто положите PDF в папку data
```bash
text data/

|--
ваш_документ_1.pdf

|--
ваш_документ_2.pdf

|--
любой_другой.pdf
```
# Перезапустите API
```bash
python main.py
```
## Что произойдёт автоматически:
1. Бот найдёт все новые PDF

2. Извлечёт текст

3. Разобьёт на чанки

4. Перестроит индекс 
5. Начнёт отвечать по новым документам

# Важно
```bash
|--1. Поддерживаются только PDF с текстовым слоем -|
|--2. Сканированные PDF (без текста) не работают --|
|--3. Можно загружать несколько файлов одновременно|
```

# Частые проблемы и их решение

## 1. Ошибка: "No module named 'fastapi'"
Решение:

```bash
pip install -r requirements.txt
```
## 2. Ошибка: "GROQ_API_KEY not found"
### Решение:

--Создать файл .env

--Добавить: GROQ_API_KEY=ваш_ключ

--Перезапустить API

## 3. Ошибка: "Model has been decommissioned"
Решение:

--Обновить модель в .env:
```bash
env
GROQ_MODEL=llama-3.1-8b-instant
```
### ***Или использовать: llama-3.1-70b-versatile (ну или более свежую версию)***
